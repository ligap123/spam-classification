二、正常短信示例
--------------

1. 验证码类
----------
【支付宝】您的验证码是123456，5分钟内有效，请勿泄露给他人。

【微信】您的验证码是789012，10分钟内有效，如非本人操作请忽略。

2. 通知类
--------
【XX快递】您的快递已送达，请凭取件码1234到驿站领取。

【XX银行】您的账户于2024年1月15日消费100.00元，余额1000.00元。

3. 日常交流类
----------
今晚有空吗？一起吃饭吧。

好的，我马上到。

会议改到下午3点，请知悉。

4. 服务提醒类
----------
【XX医院】您的预约时间是2024年1月20日上午9:00，请准时到达。

【XX物业】您的物业费已缴纳，缴费凭证已发送至您的邮箱。


三、测试建议
----------

1. 先测试明显的垃圾短信，验证模型识别能力
2. 再测试正常短信，验证模型误报率
3. 测试边界案例（如：正常短信中包含"中奖"、"免费"等关键词）
4. 对比两个模型的预测结果
5. 查看 LLM 分析报告，了解风险因素和行动建议


四、测试步骤
----------

1. 复制上述任意一条短信
2. 粘贴到 Web 界面的输入框
3. 点击"分析短信"按钮
4. 查看预测结果和 LLM 分析报告
5. 尝试切换模型，对比结果
6. 点击"模型对比"查看两个模型的一致性


五、预期结果
----------

垃圾短信应该被识别为：
- 预测结果：垃圾短信
- 概率：> 80%
- 风险因素：中奖信息、诱导点击、虚假优惠等
- 行动建议：删除、举报、忽略

正常短信应该被识别为：
- 预测结果：正常短信
- 概率：< 20%
- 风险因素：未识别到明显风险因素
- 行动建议：正常处理
